Prompt to fix alignment (HTML/CSS → print/PDF)

You are improving a one-page A4 Surgical Case Sheet built with HTML/CSS and exported to PDF (browser print). The current page has misaligned labels/values and uneven spacing.
Goal: perfect grid alignment, consistent spacing, print-ready.

What to do

Refactor layout

Use CSS Grid for the form area: 2 columns: 35% (labels) / 65% (values).

Vertically center labels and keep all label text right-aligned; value fields left-aligned.

Keep equal row gaps (e.g., 12px).

Header

Center a bold title block “SURGICAL CASE SHEET” with a subtle background.

First row under the title is a 2-column bar: Case Sheet No. (left) and Date (right) using display:flex; justify-content:space-between;.

Typography & spacing

Base font: system UI or Inter, 12–13px; titles 16–18px.

Use a tight vertical rhythm: line-height:1.35.

Consistent margins and padding.

Print / PDF

@page { size: A4; margin: 18mm; }

Use a .page wrapper sized for A4 with white background.

Avoid page breaks (single page); if needed, add page-break-inside: avoid;.

Value fields

For blank lines, render a 1px bottom border (.underline) that stretches full value column width.

Keep units like Age and Sex on the same baseline.

Accessibility & RTL safety

Semantic HTML (<label for>, <dl>, or proper form grouping).

Don’t use absolute positioning for alignment.

Deliverables

A single index.html with embedded <style> that I can open and Print to PDF.

Keep all styles in the same file.

No external fonts required.

Acceptance checklist (verify before returning)

All label/value pairs line up in a strict 2-column grid.

Title centered; “Case Sheet No.” left & “Date” right on the same row.

Every blank field shows a consistent underline across the value column.

No content shifts when pressing Ctrl/Cmd+P → A4.

No text overflow or wrapping of labels.
